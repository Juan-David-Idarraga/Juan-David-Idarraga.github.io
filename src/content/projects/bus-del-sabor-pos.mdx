## Contexto

Bus del Sabor POS es un sistema pensado para una operacion gastronomica movil, donde el acceso al software debe ser rapido, claro y conectado con la identidad del negocio.

La pantalla inicial funciona como punto de entrada antes de ingresar al flujo principal del POS. Desde ahi el usuario se autentica e inicia turno, preparando la jornada operacional antes de registrar ventas, caja o movimientos internos.

## Pantalla analizada

La primera pantalla corresponde al acceso del sistema. Incluye campos de usuario y contrasena, una llamada clara a iniciar turno y una identidad visual asociada a un food truck.

Esta imagen comunica tres elementos importantes del producto:

- el sistema tiene marca propia;
- la operacion parte desde un turno o jornada;
- existe control de acceso antes de entrar al POS.

## Criterio funcional

El inicio de turno no es solo una pantalla de login. En un sistema POS, este paso puede convertirse en la base para registrar actividad por operador, controlar caja, asociar ventas a una jornada y mantener trazabilidad administrativa.

Por eso esta pantalla se documenta como el primer bloque del flujo operacional. A medida que se agreguen nuevas capturas, el caso puede ampliarse con ventas, productos, inventario, reportes y cierre de caja.

## Evolucion del caso

Este proyecto queda preparado para incorporar nuevas pantallas del sistema. Las siguientes imagenes recomendadas para completar el caso son:

- pantalla principal de ventas;
- seleccion de productos;
- caja o resumen de pedido;
- administracion de productos;
- inventario o stock;
- reportes;
- cierre de turno o caja.
